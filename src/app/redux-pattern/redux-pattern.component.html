Redux Pattern

Redux changes flux in a way that development tools can better work with it. For example with Redux time traveling is possible.

Redux Flow

First an Action is created by a user event. Then the event is sent to a core Reducer function, that distributes the Action to its child Reducers, that process the Action inside a switch statement and return a copy of the changed state. The new state is saved to the application Store. Views subscribed to the store get notifed about state changes and re-render.

Action

An Action consists of a "type" and a "payload" that can be created by action creator methods that are grouped to action helpers. Actions can be created by users, e.g. click event, or server responses. (same as flux)

Reducer

Reducer are have the state logic inside and are responsible for handling transitions from one state to the next one. Reducers are pure functions which produce the same output for a given input. Compared to flux the switch statement lives inside Reducers in redux.

Store

Redux has only one single Store that is dumb and his responsiblity is storing the application state. Compared to flux in redux the Store has no logic.

View

Views subscribe to the state and get notified about changes by observables.
